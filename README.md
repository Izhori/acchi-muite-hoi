# あっち向いてホイ ✕ AI

カメラとマイクを使って、AIキャラクターと「じゃんけん → あっち向いてホイ」で遊べるブラウザゲームです。
顔の向き・手の形（グー / チョキ / パー）・指さしの向きを端末内でリアルタイム判定します。

- 映像・音声はすべて端末内（ブラウザ）で処理され、どこにも送信されません。
- 顔と手の認識には Google MediaPipe Tasks Vision を使用しています（CDN読み込み）。

## 遊び方

1. **ゲーム開始**を押して、カメラとマイクを許可します。
2. 開始アニメの後、**正面を向いて「オッケー」と声に出す**と基準位置が登録され、じゃんけんが始まります（ボタン操作は不要）。
3. **じゃんけん**：「最初はグー、じゃんけん、ぽん！」の「ぽん！」の瞬間に、手（✊ / ✌️ / ✋）をカメラに見せます。
4. **あなたが勝ったら**：「あっち向いて、ホイ！」と声に出し、「ホイ」で人差し指を上下左右どれかに向けます。AIの顔の向きと同じなら**あなたの勝ち🎉**。
5. **あなたが負けたら**：AIが指さす向きが出るので、顔を上下左右どれかに向けます。AIと同じ向きだと**AIの勝ち**。ズレれば続行。

## 動作環境

- 推奨ブラウザ：**Chrome / Edge / Safari（iOS）**
- カメラとマイクの使用許可が必要です。
- 明るい場所で、顔・手を大きめに見せると安定します。

## ファイル構成

- `index.html` … アプリ本体（単一HTML。ロジック・スタイルを内包）
- `manifest.json` … PWA用マニフェスト
- `icon-180.png` / `icon-192.png` / `icon-512.png` / `favicon.png` … アイコン
- `*.mp4` / `*.png` / `*.m4a` … AIキャラクターの動画・画像・音声素材
- `.nojekyll` … GitHub Pages で素材ファイルを正しく配信するための空ファイル

## GitHub Pages で公開する手順

1. GitHub で新しいリポジトリを作成します（例：`acchi-muite-hoi-ai`）。
2. このフォルダ内の**すべてのファイル**（`index.html` を含む）をリポジトリの**ルート**に置いてコミット／アップロードします。
   - Web からアップロードする場合：リポジトリの **Add file → Upload files** で、このフォルダの中身をドラッグ＆ドロップ。
3. リポジトリの **Settings → Pages** を開きます。
4. **Build and deployment** の **Source** を **Deploy from a branch** にし、Branch を `main` / フォルダを `/ (root)` にして保存します。
5. 数十秒〜数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

## iPhone（Safari）でアプリのように使う

1. 上記の公開URLを **Safari** で開きます。
2. 共有ボタン → **ホーム画面に追加** をタップします。
3. ホーム画面のアイコンから起動すると、全画面（standalone）で遊べます。
4. 初回はカメラとマイクの許可を求められます。**許可**してください。

## 注意

- `http://` や `file://` ではカメラ／マイクが使えない場合があります。**`https://`（GitHub Pages はhttps）** で開いてください。
